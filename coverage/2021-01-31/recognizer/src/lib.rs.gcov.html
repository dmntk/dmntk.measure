     293             : //! ## Algorithm of recognizing the preferred orientation of the decision table
     294             : //!
     295             : //! Algorithm of recognizing the preferred orientation of the decision table is presented
     296             : //! below as a decision table. Correctly constructed decision table may be presented
     297             : //! horizontally (rules as rows), vertically (rules as columns) or as crosstab.
     298             : //!
     299             : //! Every decision table must contain minimum one vertical and one horizontal double line.
     300             : //! If the decision table contains annotation clause, then the must be second horizontal
     301             : //! or vertical line present. When the decision table does not contain any double lines
     302             : //! or contains more than two horizontal or two vertical lines, then such decision table
     303             : //! is invalid and will not be processed. This situation is covered by rules **1** and **2**.
     304             : //!
     305             : //! Rules **3** and **4** detect **horizontal decision table**. Horizontal decision table must
     306             : //! consist of single horizontal double line, one or two vertical double lines, hit policy
     307             : //! marker must be placed in the top left corner, and the rule numbers must be placed
     308             : //! in the first left column below the horizontal double lines. When all these conditions
     309             : //! are met, decision table is oriented **horizontally** (rules as rows).
     310             : //!
     311             : //! Rules **5** and **6** detect **vertical decision table**. Vertical decision table contains
     312             : //! single vertical double line and one or two horizontal double line. When vertical decision
     313             : //! table contains annotation clause, than contains two horizontal double lines.
     314             : //! In horizontal decision table the hit policy marker must be placed in the left bottom
     315             : //! corner, and the rule numbers must be placed in the last row on the right side of the vertical
     316             : //! double line. When all these conditions are met, decision table is oriented **vertically**
     317             : //! (rules as columns).
     318             : //!
     319             : //! Rule **7** detects crosstab decision table. Crosstab decision table contains exactly one
     320             : //! vertical and one horizontal line. Crosstab decision table does not contain any hit policy
     321             : //! marker nor rule numbers. When all these conditions are met, decision table is a **crosstab**.
     322             : //!
     323             : //! ```text
     324             : //! ┌─────────────────────────┐
     325             : //! │  Preferred orientation  │
     326             : //! ├───┬────────────────────┬┴─────────────────────┬──────────────┬──────────────╥─────────────┐
     327             : //! │ F │ Number of vertical │ Number of horizontal │  Hit policy  │    Rule      ║  Preferred  │
     328             : //! │   │    double lines    │     double lines     │              │   numbers    ║ orientation │
     329             : //! │   ├────────────────────┼──────────────────────┼──────────────┼──────────────╫─────────────┤
     330             : //! │   │    <1, 1, 2, >2    │     <1, 1, 2, >2     │  top-left,   │ left-below,  ║ horizontal, │
     331             : //! │   │                    │                      │ bottom-left, │ right-after, ║  vertical,  │
     332             : //! │   │                    │                      │  not-found   │  not-found   ║  crosstab,  │
     333             : //! │   │                    │                      │              │              ║    error    │
     334             : //! ╞═══╪════════════════════╪══════════════════════╪══════════════╪══════════════╬═════════════╡
     335             : //! │ 1 │       <1, >2       │         -            │      -       │      -       ║    error    │
     336             : //! ├───┼────────────────────┼──────────────────────┼──────────────┼──────────────╫─────────────┤
     337             : //! │ 2 │         -          │       <1, >2         │      -       │      -       ║    error    │

     339             : //! │ 3 │         2          │         1            │  top-left    │ left-below   ║ horizontal  │
